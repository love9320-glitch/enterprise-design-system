**06-30 첫 적용 + textarea 예외 정립:** TextArea의 네이티브 `<textarea>`는 ScrollArea(div용 JS 오버레이)로 못 감쌈. **시도1: `.custom-scrollbar` CSS로 네이티브 바 스타일링 → hover 불안정(Firefox 불가)으로 실패(사용자 "그대로야").** **시도2: auto-grow+ScrollArea로 감싸기 → 커서 추적(caret) 깨짐(예상).** **정답(채택): 네이티브 스크롤(커서 추적) 유지 + `.hide-native-scroll`로 기본 바만 숨기고, ScrollArea와 같은 오버레이 thumb(scroll-bar 토큰·hover·드래그, ScrollArea 세로 thumb 계산 그대로 포팅)를 textarea 스크롤에 동기화해 얹음.** 규칙 9에 이 패턴을 textarea 예외로 명시, `.custom-scrollbar`는 제거. 07-06 Table bordered+maxHeight: sticky 헤더가 알파 배경이라 스크롤 행이 비침(사용자 지적) → 헤더를 스크롤 밖 분리, 바디만 ScrollArea(maxHeight) — 동일 colgroup+table-fixed로 컬럼 정렬, 가로는 바깥 ScrollArea로 헤더·바디 동반. ~~남은 한계: 가로 오버플로 시 세로 thumb가 콘텐츠 우측(뷰포트 밖)에 붙음(실사용 빈도 낮아 보류)~~ → **07-06 해소(사용자 지적): ScrollArea에 vScrollEl 신설 — 세로 thumb를 가로 스크롤을 안 받는 바깥 가로 ScrollArea가 바디 스크롤 요소에 연동해 그림(Table 헤더 분리형은 바디를 thumb 없는 hide-native-scroll 스크롤 div로 교체). 세로 thumb가 가로 스크롤 위치와 무관하게 항상 보임.** 같은 날 사용자 지시("앞으로 스크롤 생길 때 적용")로 이 중첩 패턴+thumb 규격(시각 6px·히트 8px)을 규칙 9 본문에 명문화 — 이후 가로+세로 중첩 스크롤은 vScrollEl 연동이 표준. 07-06 SideNavigation 메뉴 목록도 ScrollArea로 — **부모 유동 높이엔 뷰포트에 `max-h-full`이 핵심**(maxHeight prop 없이 flex-1 루트만으론 뷰포트가 안 줄어듦 — App.jsx absolute inset-0 방식의 flex 버전 판례). 같은 날 Table bordered 분리 바디에도 동일 패턴 적용(maxHeight='fill'+minHeight — min 생략=hug, 상한=모달 가용 높이·ModalBodyMaxContext 체인). 이후 noneline도 분리 방식으로 통일(sticky 경로 삭제 — 헤더 위로 행 스크롤 잔상 제거, fill도 두 타입 지원). 07-14 스크리닝 빌더: 카드/수식 영역을 각자 독립 ScrollArea로 분리(요구), 고정 헤더+페이드 아래에서 thumb가 시작하도록 **data-scroll-sticky-top 스페이서 재사용** — sticky 헤더용 마커를 absolute 오버레이 헤더에 응용(스페이서=헤더 높이, 레이아웃·트랙 동시 해결). 아코디언 maxHeight 본문도 ScrollArea.
